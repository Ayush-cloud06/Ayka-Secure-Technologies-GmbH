# Roles and Responsibilities

**Organization:** Ayka Secure Technologies GmbH  
**Location:** Stuttgart, Germany

## 1. Purpose

This document defines formal role-based responsibilities within Ayka Secure Technologies GmbH to ensure:

- Clear accountability for information security
- Proper governance oversight
- ISO/IEC 27001 alignment
- Effective risk ownership
- Separation of duties where feasible

Responsibilities apply regardless of whether an individual holds multiple roles.

## 2. Executive Management (Managing Director / CEO)

**Responsibilities:**

- Approves the Information Security Policy
- Accepts residual risk
- Ensures adequate security budget and resources
- Reviews ISMS performance annually
- Approves major compliance decisions
- Ensures alignment between business objectives and security posture

## 3. Chief Information Security Officer (CISO)

**Responsibilities:**

- Owns the Information Security Management System (ISMS)
- Maintains the risk register
- Oversees control implementation
- Coordinates internal audits
- Ensures ISO 27001 alignment
- Oversees incident response governance
- Reports security posture to executive management
- Reviews access control assignments

The CISO maintains independence from direct revenue targets.

## 4. Cloud Platform Engineering

**Responsibilities:**

- Designs and maintains AWS cloud infrastructure
- Implements security controls defined by the ISMS
- Maintains IAM role structures
- Ensures logging and monitoring are operational
- Supports evidence collection automation
- Responds to security remediation requests

## 5. Compliance & Risk Management

**Responsibilities:**

- Maintains GDPR documentation (ROPA, retention policies)
- Conducts periodic risk assessments
- Tracks risk treatment progress
- Maps technical controls to regulatory frameworks
- Supports audit preparation

## 6. Security Operations

**Responsibilities:**

- Monitors SIEM alerts (Microsoft Sentinel)
- Investigates security incidents
- Coordinates incident response activities
- Maintains incident documentation
- Performs periodic access reviews

## 7. Product & Engineering

**Responsibilities:**

- Ensures SaaS application security by design
- Participates in threat modeling activities
- Implements secure development lifecycle practices
- Addresses vulnerabilities identified by security teams

## 8. HR & Administration

**Responsibilities:**

- Ensures confidentiality agreements are signed
- Tracks mandatory security awareness training
- Notifies IT of onboarding and offboarding events
- Maintains personnel documentation securely

## 9. All Employees

All employees are responsible for:

- Protecting company and customer data
- Reporting security incidents immediately
- Completing annual security awareness training
- Following established security policies
